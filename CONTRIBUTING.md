# Contributing to Excanon

Thank you for your interest in contributing to Excanon! We welcome bug reports, feature requests, documentation improvements, and code contributions.

## Getting Started

1. Fork the repository.
2. Create a branch for your work: `git checkout -b my-feature`.
3. Make your changes.
4. Run the test suite.
5. Submit a pull request.

## How to Contribute

- If you find a bug, open an issue with a clear description and steps to reproduce.
- If you want to request a new feature, explain the use case and expected behavior.
- If you want to improve documentation, examples, or README content, open a pull request directly.

## Development Setup

```bash
git clone https://github.com/yourusername/excanon.git
cd excanon
mix deps.get
```

Run tests with:

```bash
mix test
```

## Code Style

Please follow standard Elixir conventions and run `mix credo` to check for linting issues. Keep diffs small and focused, and update existing tests or add new ones for any behavior changes.

## Pull Request Process

- Open a pull request against `main`.
- Provide a clear title and description of your changes.
- Mention any related issues or discussion.
- Ensure tests pass and documentation is updated as needed.

## Reporting Security Issues

If you discover a security vulnerability, please report it privately by opening an issue marked as confidential or contacting the maintainers directly.
